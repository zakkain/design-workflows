Apps of convenience

Single-task apps that do one thing very well is the way we're headed

Boom

Command-line snippets, by Zach Holman

  • No more forgetting obscure git commands
  • set up snippets for editing certain files or cd-ing to key directories
  • the more time you spend in the terminal, the more useful it is for everything

Hub

Command line wrapper for github-based git commands, by Defunkt

  • no more typing out git@github.com:user/repoName.git to clone, etc
  • fork, browse issues, wikis, launch compare views, all from the CLI
  • not sexy, but so convenient

Menu-bar Apps

  • Better Touch Tool: custom gestures, window-snapping à la Windows 7. (app store)
  • Characters: one-click copying of unicode characters and HTML entity codes. (app store)
  • Frank DeLoupe: color-picker that handles hex, RGB(A), HSL(A). (app store)
  • Bartender: hides all these menubar apps behind a single icon link

A better way

DevelopmentSeed

We began developing the majority of our projects for clients as simple, single page websites by manually coding the HTML, CSS, and JavaScript files. We pushed the boundaries of what we could do we these standards and avoided building server applications at all costs. As a result, we were able to build some of our best work.

“Embedding services for advanced functionality”

  • Static sites serving up JSON & 3rd-party APIs is flexible and nimble
  • Prose.io
  • Flickr is a better photo CMS than I could build with Wordpress custom fields
  • The downside is reliance on 3rd-party systems.

Why I like it

We’re now at a point where we can build completely client-side websites that have advanced server-side applications embedded from dedicated external services. If one service fails, it does not impact others or the content of the website. This is a marked improvement in resilience over the previous model.
  • easier for me to execute
  • more familiar for the client, and probably cheaper
